Webhook receiver

Ready for One Horizon events.

This deployment is online. Use /webhook as the endpoint URL in One Horizon. The endpoint is protected, so a browser visit can return 401.

Root page live POST /webhook
1. Set the key Add ONE_WEBHOOK_KEY to your host.
2. Match One Horizon Use the same key in Settings -> Apps.
3. Send events Subscribe to events and point them at /webhook.
CloudEvents JSON 
POST /webhook
content-type: application/cloudevents+json
x-one-webhook-key: your_webhook_key
x-one-event-type: task.created
x-one-event-id: evt_123

Return a fast 2xx response, store event IDs before side effects, and avoid logging full payloads in production.
